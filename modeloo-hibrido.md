# Modelo Híbrido: Cascada + Scrum

## 1. Metodologías Elegidas

Para el desarrollo del proyecto de monitoreo de equipos y protocolos en red, hemos seleccionado un modelo híbrido que combina dos metodologías:

- **Cascada**: para la planificación, definición de requisitos y diseño inicial.  
- **Scrum**: para la ejecución iterativa, desarrollo incremental y mejora continua.

---

## 2. Justificación de la Elección

- **Cascada** aporta una estructura clara y secuencial, ideal para definir con detalle los requisitos y la arquitectura del sistema antes de empezar el desarrollo. Esto es importante para asegurar que el proyecto tenga una base sólida y que todos los involucrados comprendan el alcance y objetivos desde el inicio.  

- **Scrum** permite trabajar de forma ágil e iterativa durante la implementación, favoreciendo la entrega de partes funcionales en cortos periodos (sprints), la adaptación a cambios y la retroalimentación constante, lo que reduce riesgos y mejora la calidad final.

Combinar ambos modelos facilita un equilibrio entre planificación y flexibilidad.

---

## 3. Aportaciones de Cada Metodología

| Metodología | Qué aporta                             |
|-------------|-------------------------------------|
| Cascada     | Claridad en requisitos y diseño inicial. Documentación detallada. Definición formal del proyecto. |
| Scrum       | Adaptabilidad, entregas parciales frecuentes, feedback continuo, mejora constante y gestión ágil del equipo. |

---

## 4. Aplicación en el Proyecto

### Fase Inicial - Cascada

- Definición clara de los requisitos del sistema de monitoreo.  
- Diseño del modelo de base de datos y estructura general.  
- Planificación global del proyecto y división en módulos.

### Fase de Desarrollo - Scrum

- División del proyecto en sprints de 2 semanas.  
- Planificación de cada sprint en reuniones de Sprint Planning.  
- Ejecución diaria con Daily Scrum para seguimiento.  
- Entregas parciales al final de cada sprint para revisión y retroalimentación.  
- Evaluación en Sprint Review y mejora continua en Sprint Retrospective.

---

## 5. Beneficios del Modelo Híbrido en Este Proyecto

- Se asegura que el proyecto tenga un fundamento sólido antes de comenzar a desarrollar.  
- Se facilita la adaptación a cambios o nuevos requerimientos durante el desarrollo.  
- Se mejora la comunicación entre el equipo y el cliente (docente).  
- Se reducen riesgos al entregar funcionalidad progresivamente.  
- Se mantiene un balance entre documentación y agilidad.

---

## 6. Conclusión

La combinación de Cascada para la fase inicial y Scrum para la ejecución ágil permite manejar este proyecto de monitoreo de equipos y protocolos en red con una estrategia robusta, adaptable y orientada a entregar valor continuo, logrando un desarrollo controlado y flexible que responde a las necesidades académicas y técnicas.
